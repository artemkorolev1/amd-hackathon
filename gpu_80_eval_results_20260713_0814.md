## Key Findings

1. **All 3 models load and run on GPU**: Each uses ~1.3-1.4 GiB VRAM, total ~4.5 GiB for all three (fits in 8 GiB A4000).

2. **Dataset has 58 tasks, not 80**: The file `factual_combined_80.json` contains only 58 items despite the "80" in the filename.

3. **Deterministic workers dominate shared pools**: When det and local workers share category pools, det workers process all tasks in microseconds, starving local models. The judge's "speed dominance fix" (waiting for local answers) only works if local models actually get tasks — which they don't when competing with det workers.

4. **Pure GPU mode works well**: With det workers disabled, all 3 local models cooperate effectively, achieving 100% non-empty answers in 76.7s for 58 tasks.

5. **23/58 tasks got empty answers in combined mode**: Det workers lack knowledge for factual/deep knowledge questions, producing degenerate answers. Local GPU models would have provided quality answers but couldn't compete for tasks.

6. **Recommended fix**: Partition category pools by worker type, or implement task reservation for slower GPU workers, so both fast deterministic solvers and GPU models can contribute without starvation.
